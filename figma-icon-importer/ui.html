Central Icons Importer

Imports icons into the current page. Duplicates are skipped automatically.

JSON format reference

The JSON file should be an object keyed by category name (must match a frame name on the current page, or a new frame is created). Each category contains icons keyed by kebab-case name. Each icon has a title (becomes the component set name) and variants (keyed by variant property string, value is the inner SVG markup).

{
  "AI & Magic": {
    "sparkle": {
      "title": "sparkle, star, magic, ai",
      "variants": {
        "filled=off, stroke=2, radius=0, join=square": "<path d=\"M12 2L...\" stroke=\"currentColor\" stroke-width=\"2\"/>",
        "filled=on, stroke=2, radius=0, join=square": "<path d=\"M12 1...\" fill=\"currentColor\"/>",
        "filled=off, stroke=2, radius=0, join=round": "<path ... stroke-linejoin=\"round\"/>",
        "...": "..."
      }
    },
    "wand": {
      "title": "wand, magic-wand",
      "variants": { "...": "..." }
    }
  },
  "Arrows": {
    "arrow-up-wall": {
      "title": "arrow-up-wall",
      "variants": { "...": "..." }
    }
  }
}

Variant keys follow the pattern:
filled=off|on, stroke=2|1.5|1, radius=0|1|2|3, join=square|round
(30 combinations: square only has radius=0, round has 0-3)

Variant values are the inner SVG content (paths, circles, rects etc.) — everything inside the <svg> tag. The plugin wraps them in a 24x24 viewBox automatically.
